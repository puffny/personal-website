# 个人作品集网站设计指南

Version 1.0

## 1. 设计关键词

整体风格：

- 现代
- 专业
- 克制
- 高级感

关键词：

- Minimal
- Professional
- Clean
- Modern
- Elegant
- Technology
- Motion

视觉参考：

- Apple
- Stripe
- Linear
- Vercel
- Raycast

## 2. 设计原则

- 大留白：保持页面呼吸感，避免信息过度拥挤。
- 弱边框：用轻量分割和层次关系组织内容。
- 强排版：通过字号、字重、行距和对齐建立专业感。
- 轻动效：使用节制的过渡、滚动渐显和 Hover 反馈。
- 内容优先：设计服务于个人能力表达和案例说服力。

## 3. 页面视觉方向

### Hero

- 第一屏需要快速建立个人品牌感。
- 可使用动态背景、品牌 Logo 和轻量动效。
- 氛围应偏科技、理性、专业。
- CTA 聚焦「下载作品集」和「联系我」。

### About & Career

- 信息结构要清晰，突出 12 年经验与完整项目链路能力。
- 职业经历建议采用时间线或分组列表。
- 当前关注方向可作为独立标签或重点短句呈现。

### Core Strengths

- 使用三张核心能力卡片承接项目案例。
- 每张卡片需要包含能力标题、说明、关键词、代表项目和跳转入口。
- 卡片视觉应清晰可扫读，避免大段文字堆叠。

### Design Thinking

- 适合采用文章摘要或观点列表形式。
- 内容表达应沉稳、真实，突出设计思考深度。
- 不需要过度装饰，重点放在文字节奏与观点层次。

### AI Lab

- 体现对 GPT、Codex、Cursor、Lovart 等工具的探索。
- 可用流程图或工具卡片呈现「设计 -> 开发 -> 部署」工作流。
- 视觉可以更轻快，但仍需保持整体专业气质。

### Contact

- 信息应直接、易联系。
- 头像、姓名、职位和联系方式需要清晰。
- 结束语：Design for People, Build for Business.

## 4. 交互与动效

基础交互要求：

- 平滑滚动
- 滚动渐显动画
- Hover 反馈
- 卡片轻微位移
- 避免复杂炫技动画

动效原则：

- 动效用于建立节奏和反馈，不抢内容注意力。
- 首屏加载要快，避免重资源阻塞。
- 移动端动效需要更克制，确保流畅和可读。

## 5. 前端实现建议

技术栈：

- React
- Vite
- CSS Modules 或 Tailwind
- Netlify 部署

组件结构：

- `Hero.jsx`
- `About.jsx`
- `CoreStrengths.jsx`
- `DesignThinking.jsx`
- `AiLab.jsx`
- `Contact.jsx`

项目详情页：

- 采用独立路由，便于后续扩展。
- `/project/tianmu`
- `/project/baozhang`
- `/project/senben`

性能要求：

- 首屏加载快速。
- 图片懒加载。
- 动效不过度。
- 移动端适配完整。

## 6. 内容表达规范

- 优先表达能力、职责、过程和结果。
- 项目案例需要突出问题、过程与成果。
- 数据表达保持克制可信，避免过度夸张。
- 文案语气应专业、清晰、自信。
